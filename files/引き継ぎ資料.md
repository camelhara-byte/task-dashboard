# ダッシュボード引き継ぎ資料
## プロジェクト概要
キャメルグループ歯科医院（錦糸町・オアシス・鎌ヶ谷）の事務担当・原さんの業務管理ダッシュボード。
院長への業務報告・可視化が目的。

## 現在の完成物
- `業務ダッシュボードv7.html`：静的HTMLダッシュボード（データ埋め込み済み）
- データソース：NotionのタスクDBをCSVエクスポートしたもの

## タブ構成（3タブ）
1. 📅 月別ログ：3月〜5月の完了案件・小タスク・進行中を月別表示
2. ⚡ アクティブ：進行中（更新日新しい順）・待ち・未着手
3. 📊 概要：KPI3枚・領域別進捗ストリップ・全タスク一覧（行色でステータス識別）

## カテゴリ（7分類）
マーケティング / マニュアル・教育 / 助成金・申請 / 院内オペレーション / 労務・人事 / システム・DX / その他

## 規模（大中小）の定義
Notionの工数レベルフィールド（大・中・小）をそのまま使用。
未設定の場合はサブタスク数で補完（10件以上=大、3-9件=中、それ以外=小）

## 次のステップ：Notion API連携（GitHub Pages）
- 方式：GitHub Actions + Notion API → 自動ビルド → GitHub Pagesで公開
- Notionを更新したら自動でダッシュボードが更新される
- 院長にURLを共有するだけでOK

## Notionデータベース情報
- データベースID：31eb5de8e03680e68d8bf4d5f38309c1
- フィールド：タスク名・ステータス・更新日・優先度・工数レベル・井田チェック・タスクの種類・担当者・期日・説明・URL

## 主要な実装メモ
- サブタスク情報はNotionのMarkdownエクスポート（Drive経由）から取得済み
- 複数リンク対応：t.links = [{label, url}, ...]
- ユーザー追加リンク：localStorage（camel_links_v1）に保存
- 進行中カードは更新日の新しい順にソート
- 月別ログ：5月→4月→3月の降順

## GitHub Actions構成イメージ
```yaml
on:
  schedule:
    - cron: '0 20 * * *'  # 毎日朝5時（JST）自動更新
  workflow_dispatch:       # 手動実行も可
jobs:
  build:
    - Notion APIからデータ取得
    - HTMLにデータ埋め込み
    - GitHub Pagesにデプロイ
```
